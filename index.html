Final Technical Verdict

Hệ thống OCR hiện tại đang bộc lộ hai lỗ hổng lớn ở hai tầng khác nhau. Ở tầng Ứng dụng (Pipeline), hệ thống đang bị lỗi routing nghiêm trọng (như thấy ở Case 2), dẫn đến rủi ro rò rỉ dữ liệu hoặc sai lệch hoàn toàn kết quả trả về. Ở tầng Engine (AI Model), công cụ có khả năng đọc ký tự (Text Extraction) xuất sắc, nhưng hoàn toàn thiếu năng lực phân tích cấu trúc tài liệu (Document Layout Analysis), dẫn đến việc gộp các cột số liệu tài chính quan trọng (Case 1), khiến dữ liệu vô dụng cho mục đích tự động hóa.

  • Ngôn ngữ: Tiếng Việt (Chính), Tiếng Anh (Phụ)
  • Độ khó OCR: 8/10 (Rất khó do nhiễu)

Điểm mạnh (Success)

  • Nhận diện chính xác 100% các con số tài chính tĩnh phức tạp (hàng nghìn tỷ).
  • Đọc rất tốt văn bản dạng đoạn văn (Thuyết minh) có khoảng cách dòng chuẩn.

Điểm yếu (Failure)

  • Thất bại trong Table Structure Recognition (Gộp cột số liệu).
  • Bị nhiễu nghiêm trọng bởi con dấu mộc đỏ và chữ ký tay.
  • Đôi khi nhận diện sai dấu câu do ảnh bị nén (blur).

Khuyến nghị cải thiện (Roadmap)

  • Pre-processing: Áp dụng Color Dropout (loại bỏ kênh màu đỏ) để xóa dấu mộc trước khi OCR.
  • Table Detection: Tích hợp các mô hình LayoutLMv3 hoặc Microsoft Table Transformer để tách bảng không viền thay vì dùng heuristic.
  • LLM Correction: Sử dụng Gemini 1.5 Pro với prompt yêu cầu "tái tạo bảng tài chính, bỏ qua nét vẽ bậy" cho bước hậu kiểm.

Khuyến nghị xử lý tức thì (Critical Priority)

  • Auditing Backend: Kiểm tra lại API Endpoint, cơ chế Message Queue (ví dụ: RabbitMQ/Kafka) để đảm bảo File URL map chính xác với Task/Job ID.
  • Sanity Check Layer: Thêm một bộ kiểm tra ngôn ngữ nhẹ (fastText) so sánh Metadata của Ground Truth và kết quả OCR trước khi lưu DB. Nếu sai lệch > 80%, đánh cờ lỗi "Mismatch" tự động.
